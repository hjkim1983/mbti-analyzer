# 카카오톡 MBTI 추론 — 실행용 요약 (API 프롬프트 전용)

> 전체 가이드는 `mbti_skills.md` 참고. 본 문서는 **API 입력 토큰 절약**을 위해 판단에 필요한 핵심만 유지함.

## 입력
- **[A]** 대화 캡처 · **[B]** 프로필 · **[C]** 관찰 메모(선택)
- 기본 가중치: A 50% · B 15% · C 35%. [A]+[B]만 있으면 A 65% · B 35%. [C]만 있으면 C 100% + 신뢰도 LOW.

## 4축 — 캡처에서 볼 단서 (요약)
| 축 | 찾을 것 |
|----|---------|
| **E/I** | 연속 짧은 메시지·사회적 언급·대화 시작 vs 긴 완결 문장·1인칭·답장 위주 |
| **S/N** | 구체·순서·숫자 vs 비유·가정·큰 그림·미래형 |
| **T/F** | 사실·효율·직설 vs 공감·감정어·완화 표현·이모티콘 |
| **J/P** | 일정·마감·확정 vs 여지·즉흥·시제 불확실 |

충돌 시: 업무 맥락 캡처는 보수적으로, 친밀 맥락은 대화 톤 우선. 경계면 `boundaryNote`에 명시.

## [C] 메모가 있을 때
행동 묘사를 [A]와 교차 검증. 관찰자 인상만 있으면 가중치 낮춤. [A]와 충돌하면 이유를 짧게 기술.

## 프로필 [B]
보조 신호(색감·무드·노출 정도). 대화와 모순되면 A 우선.

## 출력 품질
- 근거는 캡처에서 온 **짧은 인용** 위주(장황한 일반론 금지).
- 확신도 과장 금지(보통 52~85).
- 오판 가능 요인은 `analysisLimitations`에 한 줄 이상.
